# KIS · 자본시장법 컴플라이언스 조사 결과 (2026-06-29)

> ⚠️ **정보 수집 목적이며 법적 판단이 아님.** 유료 런칭 전 반드시 KIS 약관 서면확인 + 자본시장법 전문 변호사 의견 필요.
> 특히 **무등록 투자일임업은 형사처벌(자본시장법 §17, §444)** 대상이므로 임의 판단 금지.

## Bottom Line
**autofolio의 "유료 멤버십 + 알고리즘이 종목·수량·시점을 결정해 멤버 계좌에 자동주문" 모델 = 무등록 투자일임업 위험(HIGH, 형사).**
- **개인 사용(Owner가 본인 키로 본인 계좌만)** = 문제 없음(개인 자기자산 운용).
- **타인을 받는 유료 서비스** = 두 겹의 벽:
  1. **KIS 약관**: 고객 대상 서비스는 **제휴계약 필요 → "핀테크사 등 제도권금융회사가 아니면 제휴 불가"** (비제도권 차단).
  2. **자본시장법**: 제3자(앱)가 투자판단을 내려 개별계좌에 주문 = **투자일임업**. 유사투자자문업으로 회피 불가(그건 불특정다수·시그널 한정).

## 핵심 근거 (공식 출처)
- KIS 포털: 개인은 본인 명의 앱키 발급(자기 계좌 사용 OK) / 고객 대상 서비스는 "제휴법인만, **비제도권금융회사 제휴 불가**" · "시세정보 **제3자 제공 불가**". (apiportal.koreainvestment.com/about-open-api, /provider-info, /provider-process)
- 금융위 유권해석: **고객이 직접 종목·수량·가격·시기를 지정**하고 시스템은 실행만 → 투자일임/자문 아님(허용). 그러나 **투자판단의 전부/일부를 일임받아 운용 = 투자일임업**. "카피트레이딩·AI자동매매는 유사투자자문업 허용범위 외." (casenote.kr, better.fsc.go.kr)
- 대법원 2018도4413: 특정인(개별 투자목적·재산·경험 반영) 대상 투자판단 제공 = 투자자문업.
- 모의투자도 사용자별 별도 앱키 필요(공유 불가).

## autofolio에의 적용 (중요한 뉘앙스)
- 로그인 화면 카피상 autofolio는 **"제안 → 사용자 확인 후 주문 실행"**(human-in-the-loop) 설계 → 금융위 해석의 "허용" 쪽(고객이 지정·결정)에 **가까움**. 이게 유리한 점.
- 그러나 **알고리즘이 종목·수량·시점을 정하고 사용자가 클릭만** 하면, 투자판단의 주체가 앱으로 해석될 수 있어 **회색지대**. 확인 모델만으로 안전하다고 단정 불가 → 변호사 확인 필요.

## 합법 경로 옵션 (의사결정용, 택1/혼합)
1. **개인 전용** — Owner 혼자 본인 키/계좌. 즉시 합법, 게이트 없음. (기술 스택은 그대로 개인 도구로 운용 가능.)
2. **시그널/제안 전용** — 앱은 알림·제안만, **각 주문을 사용자가 직접 지정·실행**(자동주문 없음). 유사투자자문업 신고 범위 검토 가능. ※ 자동실행 빼야 함.
3. **투자일임업 등록** — 제도권 금융투자업 인가/등록(자본·인력·준법 요건 큼).
4. **혁신금융서비스(금융규제 샌드박스)** — 비제도권이 한시 허용받는 경로. 심사 ~120일, 지정 2년(연장 가능), 배상책임보험 등 요건.

## 반드시 직접 확인 (사람/전문가)
1. KIS 고객센터(1544-5000): 로그인 뒤 약관 전문 — "앱키 제3자 사용/위탁 금지", "상업적 이용" 조항 원문 + **퍼유저 자기키 위탁 모델 허용 여부** 서면 확인.
2. **자본시장법 전문 변호사**: autofolio 구조(특히 "제안→확인" 모델)가 투자일임업 해당인지 법률의견. **무등록 영업은 형사처벌.**
3. 핀테크지원센터(fintech.or.kr): 혁신금융서비스 사전상담.
4. 유사투자자문업 신고로 "시그널 전용" 운영 가능 범위.

## 기술 로드맵에의 영향
- **P2(봉투암호화)·P3(KIS 키 캡처)·배포는 계속 진행 가능** — 개인 사용 + "법적 클리어 시 즉시 가동" 기술 준비로서 유효. 멀티테넌트 flag는 계속 OFF.
- **막히는 것은 "유료로 타인을 받아 자동매매" 활성화**뿐 — 이는 위 법적 게이트 통과 후.
- 관련: [[compliance-gate-presale]] · `agents/project/MULTITENANT-FLAG-ENABLE-READINESS.md`

## 제품 포지셔닝 명확화 (Owner, 2026-06-29) — 중요
Owner 확인: autofolio = **순수 인프라/도구(SaaS)**. "회원이 **본인 계정을 직접 연동**하고, **본인이 설정한 알고리즘**에 따라 자동매매. 관리자는 회원 키를 알 필요 없고(=P2 봉투암호화와 정확히 일치), **autofolio는 제안·추천·운용을 하지 않음.** 자동화/AI 매매를 원하는 사람에게 **기반만 제공**."

→ **이 모델은 훨씬 방어 가능.** 금융위 유권해석의 **"고객이 직접 종목·수량·가격·시기를 지정하고 시스템은 실행만 = 투자일임/자문 아님(주문시스템 제공, 허용)"** 에 부합. 투자판단 주체 = **회원 본인**(autofolio 아님).

**단, "도구" 쪽에 남으려면 지켜야 할 선(자문 drift 방지):**
- autofolio가 **투자판단/추천/시그널을 제공하면** 다시 자문/일임으로 끌려감(대법원 2018도4413, albup 자동매매프로그램 판례 = 시그널 제공형은 자문 인정 사례 존재).
- ⚠️ **현 제품 모순점**: 로그인 화면/20-에이전트 시스템은 "AI가 조건을 **제안**"한다고 함 → Owner 진술("제안 안 함")과 충돌. **순수 도구 포지션을 지키려면 추천/분석 기능은 "autofolio가 제공하는 서비스"가 아니라 "회원이 직접 설정·실행하는 본인의 전략/AI"여야 함.** 이 경계를 제품·마케팅·약관에서 명확히.
- KIS 약관: "회원이 **본인 키를 본인이 쓰는 도구에 입력**"이 자기사용(허용)인지 제3자서비스(제휴필요·비제도권 불가)인지 — **KIS 직접 확인 필요**(대리입력 vs 양도 회색지대).

**남은 검증(이전보다 가벼움)**: ① 도구 포지션 유지(autofolio가 추천 주체 아님) ② KIS BYO-key 약관 확인 ③ 유료 런칭 전 자본시장법 변호사 sign-off(범위: "도구로서 자문/일임에 안 걸리는지" 확인). 무등록 투자일임 우려는 이 모델에선 크게 완화되나, 자문 drift + 판례가 fact-specific이라 변호사 확인은 여전히 권장.
