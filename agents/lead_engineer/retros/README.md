# Retrospectives

Shared retrospective records for the project. Roles link here for the post-work
reflection step (what was tried, what was learned, what changes next time).

- One retro per significant unit of work or per cycle.
- Use `TEMPLATE.md` as the starting structure.
- Keep retros short, specific, and action-oriented; a retro with no change to make
  is rare.

| File | Purpose |
| --- | --- |
| `TEMPLATE.md` | Copy this to start a new retro. |

Naming: `RETRO-<YYYY-MM-DD>-<slug>.md`.
