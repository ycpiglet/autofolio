# Retros

Role retrospectives live under `agents/{role}/retros/` when a role has a
separate session, explicit Owner request, or cycle-level need for an independent
retro.

For a single Codex session, Lead Engineer may cover role perspectives in the
cycle review instead of creating duplicate role RETRO files.

Use [TEMPLATE.md](TEMPLATE.md) when a role-specific RETRO is needed.

## Single-Session Operating Policy

- Do not create duplicate role RETRO files for routine single-session work.
- Capture role-specific evidence in the TASK, CYCLE, REVIEW, or AUDIT record.
- Create a role RETRO when the role had a separate session, owned a material
  decision, or the Owner explicitly asks for it.
