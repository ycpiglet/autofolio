Bottom Line: Unknown local ID/PW login now fails closed by default, so an unapproved person cannot self-create an owner session through the local login form.

## Signal

| Item | State | Evidence |
|------|-------|----------|
| Auto-register default | pass | `app/services/auth_service.py` |
| Dev opt-in path | pass | `AUTOFOLIO_LOCAL_AUTO_REGISTER=1` tests |
| Login UX | pass | backend 401 detail displayed in login page |
| R3 boundary | pass | no DB/payment/deploy/KIS/secret change |

## Insight

1. This removes the immediate conflict between approval-only membership and local auto-registration.
2. It is not a full membership system; signup request, deposit_pending, and Owner approval storage remain TASK-087.
3. Existing approved local users can still log in, and local/dev first-run setup remains explicit opt-in.

## Decision

1. Default local auth posture is fail-closed for unknown users.
2. Keep `AUTOFOLIO_LOCAL_AUTO_REGISTER=1` as local/dev-only bootstrap behavior.
