# Autofolio Control Desk Design

## 1) 방향성

Autofolio는 마케팅 페이지가 아니라 **실시간 거래 관제 화면**이다.
디자인 목표는 아래 4개로 고정한다.

- 빠른 의사결정: 버튼/표/알림을 1차 스캔에서 읽을 수 있어야 함.
- 위험 구간의 가시성: 실전/모의/데모를 시각적으로 즉시 구분.
- 일관된 계측값 표현: 숫자/손익/상태 텍스트의 표기와 색 규칙 통일.
- 화면 고정: 불필요한 장식 제거, 핵심 지표와 액션에 집중.

## 2) 비주얼 토큰

- 배경: 어두운 네이비 계열 페이지면(`#0b1021`) + 밝은 패널.
- 패널: 가벼운 회색-청색 톤(`#f8fafc`) 기반.
- 구분선: 옅은 파랑(`#dbeafe`) 또는 회색 중성.
- 핵심 포인트: 색상 과장보다 텍스트/아이콘/우선순위로 전달.

## 3) 한국식 손익 규칙

- 상승(양수) = **빨강** 유지.
- 하락(음수) = **파랑** 유지.
- 0 또는 중립 = **회색**.

이 규칙은 기존 계좌 KPI, 포트폴리오, 주문 로그의 색 코딩과 일치해야 한다.

## 4) 운영 모드 라벨 규칙

`KIS_ENV`를 화면 라벨로 노출할 때 다음을 강제한다.

- `mock`: Mock(데모)
- `paper`: Paper(모의투자)
- `prod`: Live(실전)
- 그 외: Unknown

각 라벨은 `app/ui/theme.py`의 helper 로 통합 관리한다.

## 5) 전개 가이드

- 변경은 기능 동작보다 우선되는 스타일 정합성 문서로 먼저 반영한다.
- 컴포넌트는 `docs/design/autofolio/COMPONENT-MATRIX.md`를 기준으로 매핑한다.
- UI 텍스트/테이블/상태 칩은 기존 `app/ui` 화면에서 점진적으로 정리한다.
